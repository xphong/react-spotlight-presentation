Component Testing

  • Systems are too coupled to be effectively unit tested
  • Integration test: data layer with UI
  • Focus on behaviour (transforming data example)
  • Avoid testing functions that call other functions
The future is actually now. We are seeing more and more people migrate. Example: Netflix, Twitter Mobile to React

Present Front-End

  • Re-use of components (DRY, SRP)
  • Decoupled architecture
  • Faster Development
  • Uni-directional flow
  • New application architectures

Everything is built in components and self-contained, allowing for re-use. Small components over big controllers (legacy apps). Don't Repeat Yourself, Single Responsibility Principle

Lousely coupled architecture - each component has its own HTML, CSS, JS. Easy to change, maintain, grow.

Faster development - HMR (explain HMR vs LiveReload, ex: working on a form), import components and modules

Uni-directional flow - Two-way binding caused problems in scalable apps .no two-way binding which makes debugging easier because it is easier to follow the flow of a function

New application architectures - SCALABLE. redux, mobX has action dispatch to store then to view. Newer ways to manage state RxJS

Previous frameworks - you never know what might come next. 2017 Could be Angular 2 or Vue.js

Getting into our comparisons...

Angular - complete rewrite of the first framework. Mostly focusing on TypeScript because of lack of ES support. Also Dart. Released Beta mode in 2015, backed by Google. Google adwords.

React - JavaScript library for building user interface. View in MVC. Pure JavaScript with JSX, templating engine. Created by Facebook. Instagram

Main Differences

Framework

Steeper learning curve

Opionated

Growing community

Library

Easy to learn, difficult ecosystem

More autonomy

Active/strong community

This is a large diference. Angular is a large framework, everything comes in the box. Whereas React itself is just a view library

Angular has a steeper learning curve because it comes with everything (router, forms, xhr tools, tests). React is easy to learn, but the ecosystem is hard. Will have to find and evaluate separate modules for use with React (Redux, thunk middleware, react-router, xhr tool)

Similar to Angular 1, Angular 2 wants you to do things a certain way. Less freedom, but consistent codebase. React gives you more choice on modules and libraries you choose to use with it.

React has a stronger community because it has been around a bit longer. Referring to the chart before, React also has a lot more people who have used it, and would use it again compared to Angular. Meaning it might have a better developer experience. Angular has a growing community, more people have been starting to use it. Azhar and I went to an Angular 2 meetup the other week, was very surprised at the amount of people.
